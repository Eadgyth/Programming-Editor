Requirements

For a programming project the computer is set up in the same way as if any text editor is used for coding and the tested program would be run from a command interpreter. Presently, programming projects can be defined for Java and Perl.

Running a project

Types of projects

The following types of projects may be defined and would be recognized based on the file extension:

  1. Java (file extension is .java)
  2. Perl (file extension is .pl, .pm)
  3. HTML (file extension is .html, .htm)

Configuring a project

  1. For a new project open a file or save a new file that has one of the file extensions mentioned above.
  2. Open the settings window by selecting 'Project' > 'Project settings'.
  3. Enter the name of the main source file without extension. The main file does not have to be the opened file but must exist.
  4. If the main file is part of a package (for Java) enter the package name in the text field 'Package containing the main class'. If it is located in a sub-package enter the path with normal file separators.
  5. If the directory that contains source files (or packages) is not supposed to be the project's root (i.e. the working directory) but a sub-directory for sources the name of this directory is entered in text field 'Name of sources directory'.
  6. A separate destination directory for compiled files (for Java) maybe entered in the text field 'Name of executables directory'. This folder will be created during compilation in the root directoy.
  7. Click ok. If entries are correct, the project can be viewed in the 'Project explorer' that is opened by selecting 'View' > 'Project explorer'.
  8. The field 'Name of project root' does not have to be filled. However, entering a name enforces that determined root directory matches this name. The text field anyway shows the name of the root when the settings window is newly opened or if a previous project is retrieved. It must be cleared (or modified) if the project root is to be changed.

Changing between projects

  • Different projects may set during a session. A new project can be set by selecting 'Project' > 'Project settings' if a file that is not part of the currently active project is in the selected tab.
  • A project inside the directory structure of an active project may be defined by selecting 'Project' > 'New settings for interlaced project' if the file in the selected tab belongs to a currently active project.
  • Changing between already set projects is done by selecting a file that is part of the project to be set active and then selecting 'Project' > 'Change project' or, to switch project and view or alter the settings, 'Project' > 'Project settings'.

Retrieving projects

  • The project that was active when the program was closed the last time is retrieved and set active when a file of that project is opened (or one is newly saved to that project) and no other project is already active. The extension of the file must correspond with the type of project (See Types of projects). The extension may be different if and only if the language is set correspondingly under 'Edit' > 'Language'.
  • If 'Save eadconfig file' is selected in the settings window the configuration of the project is stored in such config file. This is saved in the project folder but can be deleted at any time by unselecting the option to save it. A project is, in a way, "marked" as one by the presence of the eadconfig file which merely means two things:
    1. If a file of a marked project is opened (or one is saved to that project) the settings are loaded even if another project is active. Selecting 'Project' > 'Change project' can be used to change to that project or 'Project' > 'Project settings' to additionally view or alter the retrieved settings.
    2. A marked project is directly set active if a file of that project is opened and no other project is active even though that project was not necessarily active lastly when the program was closed.

Setting the language

  • The language defines the syntax coloring which firstly depends on the extension of the file that is opened or newly saved (for the file types .java, .pl, .pm, and .html).
  • Setting the language by selecting 'Edit' > 'Language' takes effect only in unnamed tabs.
  • Setting the language may also determine which input options are shown in the project settings windows. For example, a file with "wrong" extension but what is supposed to belong to a Java project (some resources file, maybe) may be just selected. Opening the settings window would show the "correct Java" options if the language was set to Java. However, the file extension that cooresponds to a project type has always the priority

Testing Java code and packing a jar file

  1. Compile the source file(s) using 'Save open files and compile' in the 'Project' menu. If errors occur it will be asked to open the console panel to view messages.
  2. If the compilation was successful the new program can be run for testing by using 'Run' in the 'Project' menu. Any output and error messages from the tested program are displayed in the console panel. In the case that the program asks for input through a command line the console area is writable. The tested program can be forcibly quit by clicking the 'stop' symbol in the toolbar of the console panel.
  3. Arguments for a start script may be entered in the text field 'Arguments' (separated by single spaces) in the settings window ('Project' > ' Project settings').
  4. A jar file may be created by selecting 'Project' > 'Create jar'. The jar file is saved in the executable's folder or, if no such folder is defined, in the project's root folder. A name for the jar file can be specified in the settings window. If no name is given the jar name is the same as the name of the main class. Note that presently only class files can be bundled in a jar.

Testing a Perl script

  1. Save changes to the script and select 'Run' in the 'Project' menu.
  2. Any output (error) is shown in the console panel. NOTE: there is a limitation when the script asks for input through a command line. In that case any output from the script expected before user input is asked for (like "Enter your name:") is not printed (more precisely it's printed only after inputs were made). The output would be shown as expected (as far as tested) if the autoflushing of Perl's STDOUT is turned on in the script itself. The other possibilty is, of course, to start the command line/terminal of the OS to test the script (see Using the console).
  3. Arguments for a start script may be entered in the text field 'Arguments' (separated by single spaces) in the settings window ('Project' > 'Project settings').
  4. The syntax coloring for Perl is presently very basic.

Viewing HTML code in a web browser

  1. Save changes to the file and select 'Run' in the 'Project' menu to view the html file in the default web browser. Any file in the root folder, not just the project file specified in the settings window, can be viewed.
  2. As for Perl code, the syntax coloring is presently rudimentary.

Using the console

The console area shall show messages after compilation of a project and the output of a program that is tested.

In addition, self-chosen system commands entered in a dialog can be run in the current working directory (the project's root directory). A run process can be forcibly quit by clicking the 'stop' symbol.

The console does not emulate the terminal of the operating system though. However, it is possible (or rather not prohibited) to run the system's terminal in the console by running the command that starts the terminal (e.g., cmd.exe in Windows). It is important to note that a tested program thereupon started from this terminal CANNOT be forcibly quit by clicking the 'stop' symbol.

Hint: using the command to start 'THE' terminal (in Windows this would be cmd.exe /c start) presets the working directory for the terminal to the project's root directory.

To run a system command...

  1. Open the console panel in the menu 'View'.
  2. Press the button 'Cmd..' and enter a command in the shown dialog window. klick ok in the dialog window to run the command.
  3. To run a previous command press the '>' button; to change a command open the command dialog again by clicking 'Cmd...'.

Further options

  • A new instance of Eadgyth can be run from the console by pressing the button with the Eadgyth icon in the toolbar of the console panel. One reason to do this would be to view the output when a plugin is tested.

Using the project explorer

The project explorer can be opened by selecting 'View' > 'Project explorer' once a project is defined.
The explorer always shows the project that is currently set active. A few operations are supported:

  • The project's file system is shown as a tree in which folders are expanded or shrunk by clicking the (+), (-) handles, as usual. However, a double left click on a folder symbol does not expand/shrink the folder but displays a new tree rooted at the clicked folder. To move back the 'Folder up' symbol is clicked. The topmost level is always the project's root directory.
  • A double left click on a file opens the clicked file in a new tab.
  • A file may be deleted after a right click on a file symbol and selecting 'delete'. If the deleted file is in an open tab, the tab is not closed but goes into 'Save as' mode.
  • A folder may be as well deleted after a right click on a folder symbol but a folder is protected from deletion if it is not empty. The exception is, if present, the folder for executable files or its subfolders. This may be deleted for a completely renewed compilation
  • A new folder may be created after a right click on a folder symbol and selecting 'Create new folder'.
  • Changes to the project's file structure made from within the program should be updated in the file view (no guarantee). Changes made from outside the program would be updated "manually" by clicking the 'refresh' symbol.
