Requirements

For a programming project the computer is simply set up in the same way as if any text editor is used for coding. Presently, programming projects can be defined for Java and Perl.

Running a project

Types of projects

The following types of projects may be defined:

  1. Java (file extension is .java)
  2. Perl (file extension is .pl, .pm)
  3. HTML (file extension is .html)

Configuring a project

The setting of a project always starts from an existing file that is opened (or newly saved) and is in the selected tab. The file is found in a directory that is supposed to be the project's root or in sub-directories of this root, e.g. a separate directory for source files.

  1. Open a file or save a new file that is of one of the file types mentioned above.
  2. Open the settings window by selecting ‘Project’ > ‘Project settings’.
  3. Enter the name of the main source file without extension. The main file does not have to be the opened file.
  4. If the main file is part of a package (applies only to Java) enter the package name in the text field ‘Package with main class’. If it is located in a sub-package enter the path with normal file separators (slash in Windows).
  5. If the directory that contains source files (or packages) is not supposed to be the project's root but a separate source directory the name of this directory is entered in text field ‘Source directory’.
  6. A separate destination directory for compiled files (applies only to Java) maybe entered in the text field ‘Executable directory’ (the folder will be created during compilation).
  7. Click ok. If entries are correct the project can be viewed in the ‘Project explorer’ that is opened in the ‘View’ menu.

Changing between projects

  • Different projects may set during a session. A new project is set by opening a file that is not part of the currently active project and then opening the settings window (‘Project’ > ‘Project settings’) to define the project's directories. The file must be found in the selected tab.
  • Changing between already set projects is done by selecting a file that is part of the project to be set active and then selecting ‘Project’ > ‘Change project’ or, to view or modify settings, ‘Project’ > ‘Project settings’.

Retrieving projects

  • The project that was active when the program was closed the last time is retrieved and set active when a file of that project is opened (or one is newly saved to that project) and no other project is active. The extension of the file must correspond with type of project (See Types of projects). The extension may be different if and only if the language is set correspondingly under ‘Edit’ > ‘Language’ before opening the file.
  • If ‘Save settings in eadconfig file’ is selected in the settings window the configuration of the project is stored in this config file. It is saved in the project folder but can be deleted at any time ("by hand" or by unchecking the option to save this file). A project is, so to say, "marked" as one by the presence of the eadconfig file which has two effects:
    1. If a file of a marked project is opened (or one is saved to that project) the settings are loaded even if another project is active. Selecting ‘Project’ > ‘Change project’ can be used to change to that project or ‘Project’ > ‘Project settings’ to additionally view or modify the configuration.
    2. A marked project is directly set active if a file of that project is opened and no other project is active even though it was not necessarily the active project when the program was closed the last time.

Setting the language

  • The language defines the syntax coloring and is internally set by the extension of the file that is opened or newly saved (for the file types .java, .pl, .pm, and .html).
  • Setting the language by selecting ‘Edit’' > ‘Language’ takes effect only in unnamed tabs.

Testing Java code and packing a jar file

  1. Compile the source file(s) using ‘Save open files and compile’ in the ‘Project’ menu. If errors occur it will be asked to open the console to view messages.
  2. If the compilation was successful the new program can be run for testing by using ‘Run’ in the ‘Project’ menu. Any output and error messages from the tested program are displayed in the console panel. The tested program can be forcibly quit by clicking the ‘stop’ symbol in the toolbar of the console panel.
  3. Arguments for a start script may be entered in the text field ‘Arguments’ (separated by single spaces) in the settings window (‘Project’ > ‘ Project settings’).
  4. A jar file may be created by selecting ‘Project’ > ‘Create jar’. The jar file is saved in the executable's folder or, if no such folder is defined, in the project's root folder. A name for the jar file can be specified in the settings window. If no name is given the jar name is the same as that for the main class. Note that presently only class files can be bundled in a jar.

Testing a Perl script

  1. Save changes to the script and select ‘Run’ in the ‘Project’ menu.
  2. Any output (error) is shown in the console panel. There is a limitation when the script asks for input through a command line: output from the perl program that normally would be send to SDOUT is not displayed immedeately (like "enter your name"). This can be circumvented by turning on the autoflushing of Perl's STDOUT in the script itself during test phase. The other possibilty is, of course, to start THE command line of the OS to test the script (see Using the console).
  3. The syntax coloring for Perl is presently very basic.

Viewing HTML code in a web browser

  1. Save changes to the file and select ‘Run’ in the ‘Project’ menu to view the html file in the default web browser.
  2. As for Perl code, the syntax coloring is presently rudimentary.

Using the console

The console area shall show messages after compilation of a project and the output of a program that is tested.

In addition, self-chosen system commands entered in a dialog can be run in the current working directory (the project's root directory). A run process can be forcibly quit by clicking the ‘stop’ symbol.

The console does not emulate the terminal of the operating system though. However, it is possible (or rather not prohibited) to run the system's terminal in the console by running the command that starts the terminal (e.g., cmd.exe in Windows). It is important to note that a tested program thereupon started from this terminal CANNOT be forcibly quit by clicking the ‘stop’ symbol.

Hint: using the command to start ‘THE’ terminal (in Windows this would be cmd.exe /c start) presets the working directory for the terminal to the project's root directory.

To run a system command...

  1. Open the console panel in the menu ‘View’.
  2. Press the button ‘Cmd...’ and enter a command in the shown dialog window. Click ok in the dialog window to run the command.
  3. To run a previous command press the ‘>’ button; to change a command open the command dialog again by clicking ‘Cmd...’.

Further options

  • A new instance of Eadgyth can be run from the console by pressing the button with the Eadgyth icon in the toolbar of the console panel. One reason to do this would be to view the output when a new plugin is tested.

Using the project explorer

The project explorer can be opened by selecting ‘View’ > ‘Project explorer’ once a project is defined.
The explorer always shows the project that is currently set active. A few operations are supported:

  • The project's file system is shown as a tree in which folders are expanded or shrunk by clicking the (+), (-) handles, as usual. However, a double left click on a folder symbol does not expand/shrink the folder but displays a new tree rooted at the clicked folder. To move back the ‘Folder up’ symbol is clicked. The topmost level is always the project's root directory.
  • A double left click on a file opens the clicked file in a new tab.
  • A file may be deleted after a right click on a file symbol and selecting ‘delete’. If the deleted file is in an open tab, the tab is not closed but goes into ‘Save as’ mode.
  • A folder may be as well deleted after a right click on a folder symbol but a folder is protected from deletion if it is not empty. The exception is, if present, the separate folder for executable files or its subfolders. This may be deleted for a completely renewed compilation
  • A new folder may be created after a right click on a folder symbol and selecting ‘Create new folder’.
  • Changes to the project's file structure made from within the program should be updated in the file view (no guarantee). Changes made from outside the program would be updated"manually" by clicking the ‘refresh’ symbol.
