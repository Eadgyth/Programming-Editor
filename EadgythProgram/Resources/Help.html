  • Requirements

    The Java runtime environment must be version JRE 8 to run Eadgyth.

    For a programming project in Java it is required that a Java development kit (JDK) is installed and the computer is set up to use the JDK. For the JDK of Oracle, see: Oracle JDK. Also, the filepath of the JDK must be saved in the ‘settings.properties’ file in the program folder. If this is not done in the file itself, the program asks to enter the filepath in a dialog the first time it is tried to compile source files or if a new version of the JDK was installed.

  • Running a Java project

    Project settings

    The setting of a project always starts from an existing file that is opened (or newly saved) and is in the selected tab. The file is found in a directory that is supposed to be the project's root or in sub-directories of this root, i.e. a separate directory for sources and/or a package.

    1. Open a java source file or save an unnamed tab as a java source file.
    2. Open the settings window using ‘Project settings’ in the menu ‘Project’.
    3. Enter the name of the source file that contains the main method in the text field ‘Name of main program file’ (without extension). The main file does not have to be the opened file.
    4. If the main file is part of a package enter the package name in the text field ‘Package with main class’. If it is located in a sub-package enter the path with normal file separators (slash in Windows).
    5. If the directory that contains source files (or packages) is not supposed to be the project's root but a separate sources directory the name of this directory is entered in text field ‘Sources directory’.
    6. A separate destination directory for compiled class files maybe entered in the text field ‘Executables directory’ (the folder will be created during compilation).
    7. Click ok. If entries are correct the project can be viewed in the ‘Project explorer’ that is opened in the ‘View’ menu.

    Compiling and testing a project

    1. Compile the source file(s) using ‘Save all and compile’ in the ‘Project’ menu. If errors occur you will be asked to open the console to view messages.
    2. If the compilation was successful the new program can be run for testing by using ‘Run’ in the ‘Project’ menu (this will also open the console panel). The tested program can be forcibly quit by clicking the ‘stop’ symbol in the toolbar of the console panel.
    3. Arguments for a start script may be entered in the text field ‘Arguments’ (separated by single spaces) in the Settings window
    4. A jar file may be created by using ‘Build’ in the ‘Project’ menu.

    Other projects than Java?

    • A project directory maybe set for the txt file type. This would be done just to view files in the project explorer but there are no actions associated with such project.
    • A project maybe also set for the html file type. HTML code can be viewed in the default browser by selecting ‘Project’ > ‘Run’. The name of the file that is viewed is entered in the text field ‘HTML file’ in the settings window. The keyword coloring for html code is rudimentary though.

    Changing between projects

    • Different projects may set during a session. A new project is set by opening a file that is not part of the currently active project and then opening the settings window (‘Project’ > ‘Project settings’). The file must be found in the selected tab.
    • Changing between already set projects is done by selecting a file that is part of the project to be set active and using ‘Project’ > ‘Change project’.

  • Using the console

    The console area shall show messages after compilation of a project and the output of a program that is tested. Interactive programs can be run too, at least Java programs.

    In addition, self-chosen system commands entered in a dialog can be run in the current working directory (a project's root directory). A run process can be forcibly quit by clicking the ‘stop’ symbol.

    Note that the console does not work as the terminal of the operating system. However, it is possible (or rather not prohibited) to run the system's terminal in the console by running the command that starts the terminal (e.g., cmd.exe in Windows). It is important to note that a tested program thereupon started from this terminal CANNOT be forcibly quit by clicking the ‘stop’ symbol.

    To run a system command...

    1. Open the console panel in the menu ‘View’.
    2. Press the button ‘Cmd...’ and enter a command in the shown dialog window. Click ok in the dialog window to run the command.
    3. To run a previous command press the ‘>’ button; to change a command open the command dialog again by clicking ‘Cmd...’.

    Further options

    • A new instance of Eadgyth can be run from the console by pressing the button with the Eadgyth icon. One reason to do this would be to view the output when a new plugin is tested.

  • Using the project explorer

    The project explorer can be opened by selecting (‘View’ > ‘Project explorer’) once a project is defined.
    The explorer always shows the project that is currently set active. A few operations are supported:

    • The project's file system is shown as a tree in which folders are expanded or shrunk by clicking the (+), (-) handles, as usual. However, a double left click on a folder does not expand/shrink the folder but displays a new tree rooted at the clicked folder. To move back the ‘Folder up’ symbol is clicked. The topmost level is the project's root directory.
    • A double left click on a file opens the clicked file in a new tab.
    • A file may be deleted after a right click on a file.
    • A folder may be as well deleted after a right click on a folder but a folder is protected from deletion if it is not empty. The exception is, if present, the separate folder for executable files or its subfolders. This may be deleted for a completely renewed compilation
    • A new folder may be created after a right click on a folder.
    • Changes to the project's file structure made from within the program should be updated in the file view (no guarantee). Changes made from outside the program can be updated ‘manually’ by clicking the ‘refresh’ symbol.
